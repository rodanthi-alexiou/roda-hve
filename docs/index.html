A living agentic-SDLC lab

Build something real, fast — and make the SDLC itself more efficient

For engineers evaluating agentic ways of working: this lab gives you a runnable app and a repeatable method you can measure. It is a growing set of scenarios — greenfield and brownfield — that I build with GitHub Copilot, then use to compare agentic SDLC methodologies (like High Velocity Engineering, or HVE) on iterations, rework, and safety. The first scenario is live: Museum Sidekick, a multimodal agent grounded in the open Met Collection API.

Why I built this

The scenario is the excuse — the method is the point

I wanted a real, running app to experiment on, not a toy. So I had the GitHub Copilot chat agent build one end to end — a greenfield POC — and now I use it as a testbed for agentic SDLC methodologies. Each increment is delivered with a different approach so I can compare, side by side, what actually makes the software development lifecycle more efficient.

Museum Sidekick is that first testbed: a working, deployed agent that plans museum tours from real, open art data. But the museum was never the point — the method is. High Velocity Engineering is the first methodology I put through its paces here; more will follow, on both greenfield builds and brownfield estates, so the lab stays a fair comparison rather than a single opinion.
