The pattern

Same accelerator, different domain

Every project on this page follows the same structure: CAIRA provides the scaffold, HVE Core drives the loop, and the domain-specific piece is just the tool layer that talks to external data. The references page maps each building block. For SDLC bottleneck playbooks, see the Playbooks page.

Reuse, don't reinvent

CAIRA reference architectures and validated IaC do the heavy lifting. You write domain logic, not boilerplate.

Grounded, not guessed

The RPI loop and the Learn MCP keep every step verified against real docs and real code.

Secure by default

Least-privilege roles, managed identity, and no baked-in keys come from the accelerator.
