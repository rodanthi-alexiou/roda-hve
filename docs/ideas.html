Why this exists

Start from what already exists, not a blank cloud

Most real work is brownfield. There is already a subscription, already some resources, already a team with opinions. The question is never "what can I build from zero," it is "how do I use the material I already have to build something secure and fast, without breaking what is running."

That is the whole thesis of this lab. Each idea takes existing Azure resources and a pile of open material (HVE Core, CAIRA, the Learn MCP, public APIs) and turns them into a working agentic app. The methodology stays the same across every idea. Only the domain changes.

Reuse, don't reinvent

Existing resources, CAIRA reference architectures, and validated IaC do the heavy lifting. You write domain logic, not boilerplate.

Grounded, not guessed

The RPI loop and the Learn MCP keep every step verified against real docs and real code, so the agent stops hallucinating and starts citing.

Secure and fast

Least-privilege roles, managed identity, and no baked-in keys come from the accelerator, so speed never costs you safety.
