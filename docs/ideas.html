Why this exists

The domain is the excuse, the method is the point

Some scenarios start from a blank page (greenfield), others plug into an estate that already exists (brownfield). Either way, the interesting question is not "can Copilot build it" — it clearly can — it is which agentic SDLC methodology delivers it best: fewer iterations, safer output, less rework.

That is the whole thesis of this lab. Each idea is a working agentic app built with Copilot and a pile of open material (HVE Core, CAIRA, the Learn MCP, public APIs), then delivered with a chosen methodology so I can compare approaches. The domain changes from card to card; what I am really testing is the process.

Reuse, don't reinvent

Existing resources, CAIRA reference architectures, and validated IaC do the heavy lifting. You write domain logic, not boilerplate — whether the estate is new or already running.

Grounded, not guessed

The RPI loop and the Learn MCP keep every step verified against real docs and real code, so the agent stops hallucinating and starts citing.

Secure and fast

Least-privilege roles, managed identity, and no baked-in keys come from the accelerator, so speed never costs you safety.
