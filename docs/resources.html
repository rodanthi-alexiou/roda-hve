Resources and references

The material behind the build

Nothing here was built from scratch. This page lists every repo, tool, and doc that went into the lab, and — more importantly — how each one was actually used. Treat it as the shopping list for your own build, greenfield or brownfield.

The three building blocks

What the whole thing stands on

These three do the most work. HVE gives you the method, CAIRA gives you the architecture, the Learn MCP keeps you honest.

ResourceWhat it isHow it was used here
microsoft/hve-core High Velocity Engineering: agents, prompts, and the RPI loop Drove every step through Research → Plan → Implement → Review instead of raw prompting
microsoft/CAIRA Reference architectures for gen-AI apps on Azure Supplied the Foundry IaC, Container Apps IaC, agent API, and React frontend — almost everything but the domain logic
Microsoft Learn MCP Live, grounded access to official Microsoft docs Verified Foundry, azd, and Container Apps APIs during Research so nothing was invented

Copilot surfaces

Where the agent actually worked

GitHub Copilot showed up in more than one place. Each surface has a different job in the loop.

Copilot Chat

The main cockpit. Ran the RPI phases, read the CAIRA skill, and called the Learn MCP tools.

Coding Agent

Handed a whole feature, went off, and opened a pull request to review. See step 6 of the walkthrough.

Copilot CLI

Optional. HVE installs as a plugin so the same loop runs from the terminal when that fits better.

Secure without slowing down

The security posture is inherited, not hand-rolled. Because the identity and role assignments come from validated CAIRA IaC, the fast path and the safe path are the same path.

Brownfield kit

For when the cloud is not empty

The flagship built into a fresh resource group — a greenfield start — but plenty of scenarios target existing estates instead. These are the pieces that help you wire into what is already running rather than starting over.

Azure MCP server

Lets Copilot query your live subscription — what resources exist, how they are configured — before it plans a single change.

Infrastructure export & IaC

Turn resources that already exist into Bicep or Terraform, so the brownfield estate becomes code you can review and extend safely.

Well-Architected guidance

Security, reliability, and cost checks to run against the existing footprint before and after you add to it.
